# Avaliação UX
O objetivo desta fase é aplicar o Questionário UEQ após a conclusão dos testes de usabilidade para capturar a percepção geral dos usuários sobre a UX do Facebook. O relatório final deve traduzir dados brutos em insights de design acionáveis, contextualizando o desempenho do sistema conforme os dados fornecidos pelo UEQ.

## Dados Coletados
<img width="600" height="405" alt="image" src="https://github.com/user-attachments/assets/4557f03f-b567-4f00-8665-7a75f5a1a190" />

A coleta de dados nesta etapa é focada na percepção e satisfação geral do usuário.
Após a execução dos testes de usabilidade, o Questionário de Experiência do Usuário (UEQ) é aplicado para quantificar a experiência.
Essas respostas são os dados brutos que foram coletadas a partir dos formúlarios submetidos aos participantes que fizeram os teste de usabilidade, permitindo calcular o desempenho do Facebook nas seis escalas de UX (Eficácia, Confiabilidade, Atração, etc.).

## Conclusão Geral

### Pontos fortes
Sistema é fácil de aprender e prático, ou seja, a usabilidade básica está consolidada.
### Pontos medianos
Em várias dimensões, o sistema é percebido como “bom, mas não excelente”. Isso indica que funciona bem, mas não encanta totalmente.

### Pontos fracos

- Falta de inovação/criatividade (Novelty);
- Baixa percepção de segurança e previsibilidade (Dependability);
- Organização pode ser melhorada;
- Atratividade estética não impressionou.

## Qualidades Pragmáticas

Essas dimensões avaliam a usabilidade prática do sistema. No gráfico, os valores estão positivos, o que demonstra boa aceitação.

### Eficiência (~0,9)
O sistema é percebido como relativamente rápido e prático, permitindo que as tarefas sejam concluídas de forma satisfatória. Apesar de positivo, não é um valor muito alto, mostrando que há espaço para otimização na agilidade e organização.

### Confiabilidade / Controlo (~0,5–0,6)
Esta dimensão é a mais fraca entre as pragmáticas. Indica que o sistema transmite alguma confiança, mas não plenamente. Os usuários podem sentir certa insegurança ou imprevisibilidade em algumas interações.

### Transparência (~1,0)
É o ponto mais forte das qualidades pragmáticas. O sistema é considerado claro e compreensível, o que favorece a curva de aprendizado. No entanto, o valor está distante do máximo (3), sugerindo que ainda existem usuários que enfrentam dificuldades.

## Qualidades Hedônicas

Essas dimensões avaliam a experiência subjetiva, ligadas a prazer, estética e inovação.

### Atratividade (~0,8)
O sistema é visto como agradável e convidativo, mas não chega a ser altamente encantador. Os usuários têm uma impressão positiva, porém moderada.

### Estimulação (~1,0)
O sistema é considerado interessante e motivador, o que contribui para o engajamento. Porém, não alcança níveis altos que gerariam entusiasmo marcante.

### Inovação (~0,2–0,3)
É a dimensão mais crítica. O sistema foi avaliado como convencional, pouco inventivo e sem diferencial inovador. Apesar de funcional, falta-lhe originalidade ou características de vanguarda que o tornem competitivo em inovação.

### Atratividade (0,83):
Com média de 0,83, a atratividade apresenta a maior pontuação entre as três categorias. Esse resultado indica que os usuários têm uma impressão geral positiva do sistema, considerando-o agradável e bom de usar. Embora não seja uma nota extremamente alta, ainda representa um ponto forte, pois demonstra que a experiência desperta aceitação e pode contribuir para o uso contínuo.

### Qualidade Pragmática (0,79):
A qualidade pragmática também aparece bem avaliada, com média de 0,79. Isso mostra que o sistema é visto como eficaz, prático e razoavelmente confiável, cumprindo sua função principal de apoiar os usuários na realização de tarefas. Esse desempenho reforça a percepção de boa usabilidade, ainda que haja espaço para melhorias em termos de eficiência e clareza.

### Qualidade Hedônica (0,31):
A qualidade hedônica foi a dimensão menos avaliada, com média de 0,31. Embora esteja no campo positivo, esse valor revela que o sistema não é percebido como inovador ou estimulante, o que pode limitar seu apelo em termos de engajamento emocional e diferenciação em relação a outras soluções.

# Gráfico de distribuição de respostas

As pontuações estão todas entre +0.09 e +1.03, o que, no UEQ, é considerado uma percepção neutra ou marginalmente positiva.

### Transparência (+1.031)

Melhor pontuação, mas moderada. O produto é minimamente compreensível e claro, mas não é intuitivo. A usabilidade básica é a menos pior, mas ainda há fricção no aprendizado.

### Eficiência (+0.938)

Moderada. O sistema é percebido como lento ou exige esforço para concluir tarefas. O fluxo de trabalho é ineficiente; os usuários gastam tempo demais para atingir seus objetivos.

### Atração (+0.833)

Moderada/Neutra. O design estético falha em agradar. A atração emocional está próxima de zero.	Os usuários não acham o produto bonito nem prazeroso de usar.

### Estimulação (+0.531)

Fraca/Neutra. O produto é visto como chato e pouco motivador. O conteúdo ou as interações não geram engajamento emocional; a experiência é passiva.

# Pontos de Falha (Pior Desempenho)

### Controle	(+0.406)

Muito Fraca/Neutra. O usuário sente pouco controle sobre o sistema. Problemas de confiabilidade e previsibilidade são grandes, levando à frustração e falta de segurança.

### Inovação	(+0.094)

Falha Total. Esta pontuação é virtualmente zero. O produto é percebido como completamente convencional, sem criatividade e desatualizado.	O produto não oferece valor de novidade ou surpresa, sendo o ponto de maior fraqueza emocional.
